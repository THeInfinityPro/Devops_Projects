# ⚠️ Challenges Faced & Solutions

During the implementation of this DevOps CI/CD project, several real-world issues were encountered and resolved:

---

## 🔴 1. GitHub Webhook Not Triggering Jenkins

**Issue:**
Jenkins job was not triggered after pushing code to GitHub.

**Cause:**
Webhook URL was incorrect or Jenkins was not publicly accessible.

**Solution:**

* Configured webhook using EC2 public IP
* Opened port 8080 in AWS Security Group
* Enabled "GitHub hook trigger for GITScm polling" in Jenkins

---

## 🔴 2. Wrong Git Repository URL in Jenkinsfile

**Issue:**
Pipeline failed with:

```
fatal: unable to access repo
```

**Cause:**
Placeholder URL (`<your-username>/repo.git`) was used.

**Solution:**
Replaced with correct repository:

```
https://github.com/THeInfinityPro/Devops-Project-2.git
```

---

## 🔴 3. Branch Mismatch (master vs main)

**Issue:**

```
Couldn't find any revision to build
```

**Cause:**
Jenkins was configured to use `master` while repo uses `main`.

**Solution:**
Updated branch to:

```
*/main
```

---

## 🔴 4. Invalid Refspec Error

**Issue:**

```
Invalid refspec refs/heads/**
```

**Cause:**
Incorrect branch format in Jenkins configuration.

**Solution:**
Changed to correct format:

```
*/main
```

---

## 🔴 5. Using Freestyle Job Instead of Pipeline

**Issue:**
Jenkins only cloned code, pipeline stages were not executed.

**Cause:**
Job type was Freestyle instead of Pipeline.

**Solution:**
Created a Pipeline job and configured Jenkinsfile execution.

---

## 🔴 6. Docker Permission Denied Error

**Issue:**

```
permission denied while trying to connect to docker.sock
```

**Cause:**
Jenkins user did not have Docker permissions.

**Solution:**

```bash
sudo usermod -aG docker jenkins
sudo systemctl restart jenkins
```

---

## 🔴 7. DockerHub Login Failed

**Issue:**

```
unauthorized: incorrect username or password
```

**Cause:**
Used DockerHub password instead of access token.

**Solution:**

* Created DockerHub Access Token
* Updated Jenkins credentials

---

## 🔴 8. Kubernetes Authentication Error

**Issue:**

```
Authentication required
```

**Cause:**
Jenkins not configured with EKS kubeconfig.

**Solution:**

```bash
aws eks update-kubeconfig --region us-east-1 --name Project
```

---

## 🔴 9. Jenkins Cannot Access Kubernetes

**Issue:**
kubectl commands failed inside pipeline.

**Cause:**
kubeconfig not available for Jenkins user.

**Solution:**

```bash
sudo cp -r ~/.kube /var/lib/jenkins/
sudo chown -R jenkins:jenkins /var/lib/jenkins/.kube
```

---

## 🔴 10. Application Not Accessible via LoadBalancer

**Issue:**

```
ERR_EMPTY_RESPONSE
```

**Cause:**
Port mismatch (container running on 80, service using 3000).

**Solution:**
Updated:

* `containerPort: 80`
* `targetPort: 80`

---

## 🔴 11. Accessing Wrong LoadBalancer

**Issue:**
Opened AWS Classic LoadBalancer instead of Kubernetes Service LB.

**Solution:**
Used:

```
kubectl get svc
```

and accessed correct EXTERNAL-IP.

---
